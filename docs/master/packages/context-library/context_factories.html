3.6. Context Factories

Context Library has a concept of Context Factory (a precise name should actually be Context Manager Factory). A Context Factory is an object which takes a value and returns a new instance of ContextManagerInterface, appropriate for given value, or null if it won’t handle the value. For example, the DefaultContextFactory creates ResourceContextManager for any PHP resource and TrivialValueWrapper for any other value (except for values that are already instances of ContextManagerInterface).

Context Library has a single stack of (custom) Context Factories (ContextFactoryStack). It’s empty by default, so initially only the DefaultContextFactory is used to generate Context Managers. A custom factory object can be pushed to the top of the stack to get precedence over other factories.

Creating custom Context Factories

A simplest way to create new Context Factory is to extend the AbstractManagedContextFactory. The new context factory must implement the getContextManager() method. The AbstractManagedContextFactory is either a Context Factory and Context Manager. When an instance of AbstractManagedContextFactory is passed to with(), it gets pushed to the top of ContextFactoryStack when entering context and popped when exiting (so the new Context Factory works for all nested contexts).

Example with custom Managed Context Factory

In the following example we’ll wrap an integer value with an object named MyCounter. Then, we’ll create a dedicated Context Manager, named MyCounterManger, to increment the counter when entering a context and decrement when exiting. Finally, we’ll provide Context Factory named MyContextFactory to recognize MyCounter objects and wrap them with MyCounterManager.

For the purpose of example we need the following symbols to be imported

1use function Tailors\Lib\Context\with;
2use Tailors\Lib\Context\AbstractManagedContextFactory;
3use Tailors\Lib\Context\ContextManagerInterface;

Our counter class will be as simple as

1final class MyCounter
2{
3    public int $value;
4
5    public function __construct(int $value)
6    {
7        $this->value = $value;
8    }
9}

The counter manager shall just increment/decrement counter’s value and print short messages when entering/exiting a context.

 1final class MyCounterManager implements ContextManagerInterface
 2{
 3    public MyCounter $counter;
 4
 5    public function __construct(MyCounter $counter)
 6    {
 7        $this->counter = $counter;
 8    }
 9
10    public function enterContext(): MyCounter
11    {
12        $this->counter->value ++;
13        print("MyCounterManager::enterContext()\n");
14        return $this->counter;
15    }
16
17    public function exitContext(?\Throwable $exception = null) : bool
18    {
19        $this->counter->value --;
20        print("MyCounterManager::exitContext()\n");
21        return false;
22    }
23}

Finally, comes the Context Factory.

 1final class MyContextFactory extends AbstractManagedContextFactory
 2{
 3    public function getContextManager($arg) : ?ContextManagerInterface
 4    {
 5        if($arg instanceof MyCounter) {
 6            return new MyCounterManager($arg);
 7        }
 8        return null;
 9    }
10}

We can now push an instance of MyContextFactory to the factory stack. To push it temporarily, we’ll create two nested contexts (outer and inner), pass an instance of MyContextFactory to the outer context and do actual job in the inner context.

1with(new MyContextFactory(), new MyCounter(0))(function ($cf, $cnt) {
2    echo "before: " . $cnt->value . "\n";
3    with($cnt)(function ($cnt) {
4        echo "in: " . $cnt->value . "\n";
5    });
6    echo "after: " . $cnt->value . "\n";
7});

It must be clear, that MyContextFactory is inactive in the outer with() (line 1). It only works when entering/exiting inner contexts (line 3 in the above snippet).

Following is the output from our example

1before: 0
2MyCounterManager::enterContext()
3in: 1
4MyCounterManager::exitContext()
5after: 0
