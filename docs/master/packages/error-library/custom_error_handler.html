4.4. Custom Error Handlers

With Error Library and Contexts you can easily use your own functions as error handlers for particular calls. A class named ErrorHandler serves the purpose.

Simple example with custom error handler

The example uses the following symbols

1use function Tailors\Lib\Context\with;
2use function Tailors\Lib\Error\errorHandler;

Our error handler will respond differently to user warnings and notices. Warnings will be printed to stderr, while notices will go normally to stdout.

1function watch(int $severity, string $message, string $file, int $line) : bool
2{
3    if ($severity & (E_USER_WARNING|E_USER_ERROR)) {
4        fprintf(STDERR, "beware, %s!\n", $message);
5    } else {
6        fprintf(STDOUT, "be cool, %s\n", $message);
7    }
8    return true;
9}

The ErrorHandler object is used to enable our error handler temporarily, so it is active only within the context

1with(errorHandler(watch::class))(function ($eh) {
2    @trigger_error('the weather is nice', E_USER_NOTICE);
3    @trigger_error('rain is coming', E_USER_WARNING);
4});
5@trigger_error('good night', E_USER_NOTICE);

The outputs from the above example are

  • stdout:

1be cool, the weather is nice

  • stderr:

1beware, rain is coming!

Note, that the last call to @trigger_error() (line 5) didn’t output anything.
