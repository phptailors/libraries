4.1. Installation

The error library is split into two packages

  • phptailors/error-interface - interface,

  • phptailors/error-library - implementation.

To install error library as a runtime dependency, type:

1php composer.phar require "phptailors/error-library:dev-master"

4.2. Purpose

The main purpose of Error library is to provide handy utilities to control the flow of PHP errors within an application. It is designed to play nicely with our Context Library, so one can temporarily redirect PHP errors to custom handlers in one location without altering other parts of code.

4.3. Basic Example

In the following example we’ll redirect errors from one invocation of a problematic function to a no-op error handler. The example uses the following functions

1use function Tailors\Lib\Context\with;
2use function Tailors\Lib\Error\empty_error_handler;

and the problematic function is

1function trouble()
2{
3    trigger_error('you have a problem');
4}

The function could normally cause some noise. For example, it could call default or an application-wide error handler. Invoking it with @ only disables error messages, but the handler is still invoked. We can prevent this by temporarily replacing original handler with the empty handler. This is easily achieved with Contexts and EmptyErrorHandler.

1with(empty_error_handler())(function ($eh) {
2    trouble();
3});
